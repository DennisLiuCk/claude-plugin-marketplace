# 學習風格插件

此插件將未發布的學習輸出風格與解釋性功能結合為 SessionStart 鉤子。

**注意：** 此插件與原始未發布的學習輸出風格不同，它還整合了 [explanatory-output-style 插件](https://github.com/anthropics/claude-code/tree/main/plugins/explanatory-output-style) 的所有功能，同時提供互動學習和教育性見解。

警告：除非您可以接受此插件的額外指令和學習模式的互動性質所產生的令牌成本，否則請勿安裝此插件。

## 功能

啟用後，此插件會在每個會話開始時自動添加指令，鼓勵 Claude：

1. **學習模式：** 通過在決策點請求有意義的代碼貢獻來讓您參與主動學習
2. **解釋性模式：** 提供關於實現選擇和代碼庫模式的教育性見解

Claude 不會自動實現所有內容，而是會：

1. 識別您可以編寫 5-10 行有意義代碼的機會
2. 專注於您的輸入真正重要的業務邏輯和設計選擇
3. 為您的貢獻準備上下文和位置
4. 解釋權衡並指導您的實現
5. 在編寫代碼之前和之後提供教育性見解

## 工作原理

該插件使用 SessionStart 鉤子將額外的上下文注入到每個會話中。此上下文指示 Claude 採用互動教學方法，您主動參與編寫代碼的關鍵部分。

## Claude 何時請求貢獻

Claude 會要求您編寫代碼：
- 具有多種有效方法的業務邏輯
- 錯誤處理策略
- 算法實現選擇
- 數據結構決策
- 用戶體驗決策
- 設計模式和架構選擇

## Claude 何時不請求貢獻

Claude 將直接實現：
- 樣板或重複代碼
- 沒有有意義選擇的明顯實現
- 配置或設置代碼
- 簡單的 CRUD 操作

## 示例互動

**Claude：** 我已經設置了身份驗證中間件。會話超時行為是安全性與用戶體驗之間的權衡 - 會話是否應在活動時自動延長，還是應有硬超時？

在 `auth/middleware.ts` 中，實現 `handleSessionTimeout()` 函數以定義超時行為。

考慮：自動延長可改善用戶體驗，但可能使會話保持更長時間開放；硬超時更安全，但可能讓活躍用戶感到沮喪。

**您：** [編寫 5-10 行實現您首選的方法]

## 教育性見解

除了互動學習，Claude 還將使用以下格式提供關於實現選擇的教育性見解：

```
`★ 見解 ─────────────────────────────────────`
[關於代碼庫或實現的 2-3 個關鍵教育要點]
`─────────────────────────────────────────────────`
```

這些見解專注於：
- 針對您代碼庫的特定實現選擇
- 代碼中的模式和約定
- 權衡和設計決策
- 特定於代碼庫的詳細信息，而非一般編程概念

## 用法

安裝後，插件會在每個會話開始時自動激活。無需額外配置。

## 從輸出風格遷移

此插件將未發布的"學習"輸出風格與已棄用的"解釋性"輸出風格結合在一起。它提供互動學習體驗，您在有意義的決策點主動貢獻代碼，同時還接收關於實現選擇的教育性見解。

如果您以前使用過 explanatory-output-style 插件，此學習插件包括所有該功能以及互動學習功能。

此 SessionStart 鉤子模式大致相當於 CLAUDE.md，但它更靈活，允許通過插件分發。

## 管理更改

- 禁用插件 - 在您的設備上保留代碼
- 卸載插件 - 從您的設備中刪除代碼
- 更新插件 - 創建此插件的本地副本以個性化它
  - 提示：要求 Claude 閱讀 https://docs.claude.com/en/docs/claude-code/plugins.md 並為您設置！

## 理念

通過實踐學習比被動觀察更有效。此插件將您與 Claude 的互動從"觀察和學習"轉變為"構建和理解"，確保您通過親手編碼有意義的邏輯來發展實踐技能。

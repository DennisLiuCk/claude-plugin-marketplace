# 解釋性輸出風格插件

此插件將已棄用的解釋性輸出風格重新創建為 SessionStart 鉤子。

警告：除非您可以接受此插件的額外指令和輸出所產生的令牌成本，否則請勿安裝此插件。

## 功能

啟用後，此插件會在每個會話開始時自動添加指令，鼓勵 Claude：

1. 提供關於實現選擇的教育性見解
2. 解釋代碼庫模式和決策
3. 平衡任務完成與學習機會

## 工作原理

該插件使用 SessionStart 鉤子將額外的上下文注入到每個會話中。此上下文指示 Claude 在編寫代碼之前和之後提供簡短的教育性解釋，格式為：

```
`★ 見解 ─────────────────────────────────────`
[2-3 個關鍵教育要點]
`─────────────────────────────────────────────────`
```

## 用法

安裝後，該插件會在每個會話開始時自動激活。無需額外配置。

見解重點關注：

- 針對您代碼庫的特定實現選擇
- 代碼中的模式和約定
- 權衡和設計決策
- 特定於代碼庫的詳細信息，而非一般編程概念

## 從輸出風格遷移

此插件替換了已棄用的"解釋性"輸出風格設置。如果您以前使用：

```json
{
  "outputStyle": "Explanatory"
}
```

您現在可以通過安裝此插件來實現相同的行為。

更一般地說，此 SessionStart 鉤子模式大致相當於 CLAUDE.md，但它更靈活，允許通過插件分發。

注意：涉及軟件開發之外任務的輸出風格，更適合表示為[子代理](https://docs.claude.com/en/docs/claude-code/sub-agents)，而不是 SessionStart 鉤子。子代理更改系統提示，而 SessionStart 鉤子添加到默認系統提示。

## 管理更改

- 禁用插件 - 在您的設備上保留代碼
- 卸載插件 - 從您的設備中刪除代碼
- 更新插件 - 創建此插件的本地副本以個性化此插件
  - 提示：要求 Claude 閱讀 https://docs.claude.com/en/docs/claude-code/plugins.md 並為您設置！

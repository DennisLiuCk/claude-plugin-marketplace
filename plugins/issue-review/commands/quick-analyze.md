# /quick-analyze - 快速問題分析

你將執行快速的問題分析，直接定位問題原因。

## 問題描述

$ARGUMENTS

## 快速分析流程

### 步驟 1：資訊提取

從問題描述中提取：
- 錯誤訊息或異常
- 相關檔案或函式名稱
- 問題現象

### 步驟 2：快速搜尋

使用 Grep 和 Glob 搜尋：
1. 搜尋錯誤訊息關鍵字
2. 定位相關檔案
3. 閱讀關鍵程式碼

### 步驟 3：快速診斷

基於搜尋結果：
1. 識別最可能的原因（1-2 個）
2. 提供簡短的修復建議

### 輸出格式

```markdown
## 🔍 快速分析結果

**問題**：[問題描述]

**可能原因**：
1. [原因 1] - 位置：`file:line`
2. [原因 2] - 位置：`file:line`

**建議修復**：
[簡短的修復建議]

**需要深入分析？** 使用 `/analyze-issue` 進行完整的三階段分析。
```

現在開始快速分析。

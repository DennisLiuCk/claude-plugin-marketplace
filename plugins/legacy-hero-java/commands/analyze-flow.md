# 分析業務流程

分析包含特定關鍵字的完整業務流程，追蹤跨多個 Service 的調用鏈。

## 使用方式

```bash
/analyze-flow 訂單處理
/analyze-flow 使用者認證
/analyze-flow 支付流程
/analyze-flow 庫存管理
```

## 分析內容

1. **流程識別**：找出相關的 Controller、Service、Repository
2. **完整追蹤**：追蹤從 API 到資料庫的完整流程
3. **跨服務調用**：分析多個 Service 間的互動
4. **事務邊界**：識別事務的開始和結束
5. **非同步操作**：識別 @Async 和事件處理
6. **異常處理**：分析異常處理和補償邏輯

## 輸出

詳細的業務流程分析報告，包含：
- 完整的業務流程圖
- 每個步驟的詳細說明
- 服務調用時序圖
- 事務和資料一致性分析
- 關鍵邏輯詳解

**注意**：所有分析經過文件審核，確保正確性。
